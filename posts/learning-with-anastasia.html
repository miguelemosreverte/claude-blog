Learning with Anastasia

A multilingual wildlife magazine for children,
illustrated by DALL-E and built to teach without condescending

Act I

The Magazine Rack

Anastasia is learning about the world. She's curious about polar bears, deep-sea creatures with iron shells, seals that live in colonies, beavers that build dams, and sea otters that float on their backs cracking shellfish. She speaks English and Spanish and is learning Russian. She deserves a magazine that meets her where she is: multilingual, beautiful, and honest about the science.

Learning with Anastasia is a collection of interactive wildlife magazines served as a static website. Each issue covers one animal, with educational content written in three languages. The artwork is generated by DALL-E 3 in a hybrid style that blends Studio Ghibli's warmth, Pixar's lighting, and Van Gogh's expressive brushwork. The result is imagery that is neither photorealistic nor cartoonish—it occupies the space where children's book illustration meets fine art.

There are five issues: Polar Bears & Antarctica, the Iron Snail (Chrysomallon squamiferum—an animal that grows a shell made of iron sulfide at hydrothermal vents), Seals of the World, Beavers as Nature's Engineers, and Sea Otters. The topics were chosen for wonder. A child who learns about an animal that armors itself in metal will remember it.

Act II

The Image Pipeline

The illustrations are the heart of the project. Each issue has a cover image and a hero panorama, generated by a Node.js script that calls the OpenAI DALL-E 3 API with carefully composed prompts. The prompts encode a specific artistic directive: combine Ghibli's 2D character personality, Pixar's global illumination, Van Gogh's sky brushstrokes, and children's book narrative warmth. 

// The artistic style directive
"A unique hybrid style combining:
  - Studio Ghibli: whimsical 2D character personality
  - Pixar: global illumination and realistic textures
  - Van Gogh: expressive brushstrokes in skies and water
  - Children's Book: narrative warmth and storytelling"

// Generation: 6 images, ~$0.36 total
node generate-images.js
node generate-images.js --missing   // only broken ones
node generate-images.js --specific polar-bears-cover

The script supports targeted regeneration. --missing regenerates only broken or missing images. --specific regenerates a single named image. This matters because DALL-E output is nondeterministic—you don't always get what you asked for on the first try. Being able to regenerate one image without touching the others lets you curate iteratively. Six HD images cost $0.36. The economics of AI-generated illustration have reached the point where a parent and child can produce a professional-looking magazine for less than a dollar.

Act III

Three Languages

The language selector sits in the top-right corner: English, Spanish, Russian. Switching is instant—no page reload, just a DOM swap. The selection persists across sessions via localStorage. The content is the same in all three languages, written at a level that respects the child's intelligence without overwhelming it.

The multilingual aspect is personal. It's built for a specific child who lives between languages. The Russian is there because that's the language of grandparents. The Spanish is there because that's home. The English is there because that's the world. The magazine doesn't teach languages explicitly—it simply exists in all three, normalizing the experience of switching between them.

The best educational software doesn't feel like school.
It feels like a magazine you'd actually want to read.

Act IV

The Stack

The project is deliberately simple. Static HTML, CSS, and JavaScript. No React. No build step. No server. Each magazine issue is a self-contained directory with its own index.html and assets/images/ folder. The landing page is a dark-themed grid of magazine cards with gold accents, hover animations, and a search bar with relevance ranking.

Deployment is a git push. GitHub Pages serves it. No CI pipeline, no Docker, no infrastructure. The setup script (./setup.sh) verifies your OpenAI API key, creates directories, generates all images, and the site is ready. The entire project—from git clone to live URL—takes less time than it takes to read this paragraph.

The simplicity is the point. This project was built with a child in mind. A child who might one day look at the source code and understand it. HTML she can read. Images she helped choose. Languages she speaks. There's no framework to learn first. It's just a website.
